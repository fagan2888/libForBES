Detailed Description

LDL factorization of A'A+bI.

Version
version 0.1
Date
Created on November 5, 2015, 1:12 AM
Author
Pantelis Sopasakis

Let \(A\in\mathbb{R}^{n\times m}\) be a non necessarily square matrix. This class, which is derived from the abstract class FactoredSolver, decomposes the matrix \(F=AA^{\top}+\beta I\) for a given scalar \(\beta\) without the need to pre-compute \(F\).

In case \(A\) is a short matrix (it has more columns than rows), then we compute \(AA^{\top}+\beta I\) and we delegate its factorization to LDLFactorization.

If \(A\) is a tall matrix (it has more rows than columns) then, using the Woodbury matrix identity we have

\[ (AA^{\top}+\beta I)^{-1} = \beta^{-1}I - \beta^{-1}A(\beta I + A^{\top}A)^{-1}A^{\top}. \]

Notice that matrix \(\tilde{F}=\beta I + A^{\top}A\) is of smaller size than \(F\). The above can be concisely written as

\[ F^{-1} = \beta^{-1}(I-A\tilde{F}^{-1}A^{\top}) \]

Then, to compute a \(z\in\mathbb{R}^{n}\) which solves

\[ Fz = t, \]

we simply do \(z=\beta^{-1}(I-A\tilde{F}^{-1}A^{\top})t\) or

\[ z = \beta^{-1}(t+Ac), \]

where \(c\) is the solution of the following system

\[ \tilde{F} c = A^{\top} t, \]

which is determined using the LDL factorization of \(\tilde{F}\).

This class is powered by SuiteSparse for sparse matrices.

Attention
In case the matrix to be factorized is dense and tall (i.e., it has more rows than columns), it is imperative that the reference passed in the constructor of S_LDLFactorization remains available when solve is called.
So, for instance, the following code would throw an error:
// create a tall dense matrix:
size_t n = 500;
size_t m = 10;
Matrix * A = new Matrix(n, m, Matrix::MATRIX_DENSE);
// construct a S_LDLFactorization object
double beta = 0.690;
FactoredSolver * solver = new S_LDLFactorization(A, beta);
int status;
status = solver -> factorize();
// delete A
delete A;
Matrix b(n,1);
Matrix x;
// The solver cannot access A
status = solver -> solve(b, x);
Note that S_LDLFactorization does not keep a hard copy of the matrix object that is passed to it, but just a reference.
There are certain more subtle cases when this may happen such as the following
class MyClass {
public:
MyClass(){
// create a tall dense matrix:
size_t n = 500;
size_t m = 10;
// Matrix A has the scope of this constructor
Matrix A(n, m, Matrix::MATRIX_DENSE);
double beta = 1.3523;
// m_solver will hold a reference to A which in turn which
// will go out of scope after the constructor returns
m_solver = new S_LDLFactorization(A, beta);
int status = m_solver -> factorize();
}
void myMethod(){
Matrix b = ...;
Matrix x = ...;
m_solver -> solve(b, x);
}
private:
FactoredSolver * m_solver;
}
A solution to the above pathological case is to store matrix objects in a way that remain accessible from the S_LDLFactorization object when solve is invoked.
