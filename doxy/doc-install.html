Installation Guide

Installation guide

Libforbes comes with an installation script (install.sh) which you can use to setup the library automatically.

However, if for any reason you would like to compile it manually, you will first need to install and setup the following dependencies:

Dependencies

Libforbes depends on the following third-part libraries:

  1. openblas
  2. SuiteSparse
  3. BLAS, lapack and lapacke

To build libforbes you will need:

  1. make
  2. g++ compiler
  3. wget (optional)

Install libraries from repos

Certain libraries can be installed using apt. Let us first install BLAS and LAPACK

sudo apt-get -y install libblas-dev checkinstall
sudo apt-get -y install liblapack-dev checkinstall

We need to download and install openblas and then create a symbolic link of libopenblas.a in /usr/lib for it to be discoverable.

sudo apt-get -y install libopenblas-base checkinstall
if [ ! -e "/usr/lib/libopenblas.a" ]; then
if [ -r "/usr/lib/openblas-base/libopenblas.a" ]; then
sudo ln -s /usr/lib/openblas-base/libopenblas.a /usr/lib/;
fi
fi

Install SuiteSparse

We will download and compile SuiteSparse in the libs subfolder of libforbes:

mkdir -p libs
wget http://faculty.cse.tamu.edu/davis/SuiteSparse/SuiteSparse-4.4.6.tar.gz -O ./libs/SuiteSparse.tar.gz
tar xvf ./libs/SuiteSparse.tar.gz -C ./libs
rm -rf ./libs/SuiteSparse.tar.gz
cd ./libs/SuiteSparse
make
cd ../../

Install Lapacke

In some systems, we may install lapacke using apt. Since, however, this is not always the case, this is how we may proceed:

wget http://www.netlib.org/lapack/lapack-3.6.0.tgz -O ./libs/lapack-3.6.0.tgz
tar zxf ./libs/lapack-3.6.0.tgz -C ./libs
rm ./libs/lapack-3.6.0.tgz
cd ./libs/lapack-3.6.0/
cp make.inc.example make.inc
cd LAPACKE
make
sudo cp ./include/lapacke.h /usr/include/
sudo cp ./include/lapacke_mangling.h /usr/include/
sudo cp ./include/lapacke_utils.h /usr/include/
cd ..
sudo cp ./liblapacke.a /usr/lib
cd ../../

Configure libforbes

Having installed all dependencies we now need to configure libforbes.

If you have used the installation script install.sh you don't need to provide any further configuration.

Otherwise, you can edit file config.mk to modify some compilation parameters.

This is an example of how file config.mk should look like:

# Path to your SuiteSparse directory
SS_DIR = /home/chung/Documents/MATLAB/SuiteSparse
# Extra headers and libraries paths
IEXTRA = /usr/include
LEXTRA = /usr/lib

Compile libforbes

To compile libforbes you can simply use the Makefile provided; simply run:

make

To do a clean make, run:

make clean all

To compile the library and all tests, run:

make test

To get help about make, run:

make help

Use libforbes

To use libforbes

  1. Include the header file ForBES.h
  2. Link to the libforbes static library (libforbes.a) using -lforbes
  3. Link to the above third-party dependencies which are necessary for libforbes to work.
