Detailed Description

A ForBES function.

Version
version 0.2
Date
Created on July 9, 2015, 3:35 AM
Author
Pantelis Sopasakis

This is a generic API for ForBES functions. Constructors for this class are protected. Instances of this class can be created using any of its subclasses a list of which can be found here.

ForBES functions are all assumed to follow be \(f:X \to \mathbb{R}\cup \{+\infty\}\), where \(X\) is a vector space, typically either \(\mathbb{R}^n\) or \(\mathbb{R}^{n\times m}\).

In general, a Function in libforbes is a tuple \((f(\cdot),\nabla f, f^*(\cdot), \nabla f^*(\cdot), \mathrm{prox}_{\gamma f}(\cdot), f(\mathrm{prox}_{\gamma f}(\cdot)))\) where some of these components are available and implemented.

Certain functions furhter allow Hessian-vector products, i.e., computation of terms of the form \(\langle \nabla^2 f(x), z\rangle\) and/or with the gradient of their conjugate \(\langle \nabla^2 f(x), z\rangle\).

Input and output arguments to Function calls are instances of Matrix.

For example, in Quadratic, which is a subclass of Function, the first four of these components are available, namely \((f(\cdot),\nabla f, f^*(\cdot), \nabla f^*(\cdot))\).

In ElasticNet, only the triplet \((f(\cdot),\mathrm{prox}_{\gamma f}(\cdot), f(\mathrm{prox}_{\gamma f}(\cdot)))\) is available.

virtual FunctionOntologicalClass Function::category ( )
pure virtual

Ontological categorization of the function.

The category of a Function defines which methods are implemented, that is, it specifies whether one can compute

  1. \(f(x)\),
  2. \(f^*(x)\),
  3. \(\nabla f(x)\),
  4. \(\nabla f^*(x)\),
  5. \(\mathrm{prox}_{\gamma f}(v)\) and \(f(\mathrm{prox}_{\gamma f}(v))\)
  6. \(\langle \nabla^2 f(x), z\rangle\) and
  7. \(\langle \nabla^2 f^*(x), z\rangle,\).
Returns
Ontological class of the current Function object.

Implemented in SeparableSum, SumOfNorm2, ConjugateFunction, DistanceToBall2, Quadratic, DistanceToBox, QuadraticLossOverAffine, IndBox, IndPos, QuadOverAffine, ElasticNet, QuadraticLoss, Norm1, Norm2, IndBall2, HingeLoss, IndProbSimplex, LogLogisticLoss, HuberLoss, Norm, IndSOC, and LQCost.

int Function::hessianProduct ( Matrix &  x,
Matrix &  z,
Matrix &  Hz 
)
virtual

Method hessianProduct computes the inner product \(\langle \nabla^2 f(x), z\rangle,\) between a vector \(z\) and the Hessian of \(f\) computed at a point \(x\).

If this method is not implemented in the derived class of Function, then an approximation is return based on the finite difference scheme

\[ \langle \nabla^2 f(x), z\rangle \simeq \frac{1}{\epsilon}[\nabla f(x+\epsilon z) - \nabla f(x)], \]

where \(\epsilon=10^{-8}\) (a small tolerance).

If the gradient of \(f\) is not available, or it fails to be computed, then, an appropriate status code will be returned.

Parameters
xpoint where the Hessian \(\nabla^2 f(x)\) is computed
zvector with which the product is computed
HzThe result
Returns
status code which is equal to STATUS_OK=0 if the computation has succeeded without any problems, STATUS_UNDEFINED_FUNCTION=2 if this function is not defined by the derived class and STATUS_NUMERICAL_PROBLEMS=1 if some numerical problems prevented the computation of a reliable result. Custom implementations are allowed to return other non-zero error/warning status codes.

Reimplemented in Quadratic, QuadraticLoss, and LogLogisticLoss.
